Pengantar Visi Komputer Lanjutan

1. Pendahuluan Visi Komputer

1.1 Definisi Visi Komputer

Visi komputer (computer vision) adalah cabang ilmu komputer yang berfokus pada pengembangan metode dan algoritma agar komputer mampu memperoleh, memproses, menganalisis, dan memahami informasi visual dari citra dan video. Tujuan akhirnya adalah menghasilkan keputusan atau pemahaman tingkat tinggi, menyerupai sistem penglihatan manusia.

Perbedaan utama:

  • Pengolahan Citra (Image Processing) → fokus pada transformasi atau perbaikan citra
  • Visi Komputer (Computer Vision) → fokus pada pemahaman isi citra (objek, pola, makna)

1.2 Tahapan dalam Visi Komputer (Penjelasan Rinci)

Tahapan dalam visi komputer membentuk sebuah pipeline pemrosesan visual. Setiap tahap saling bergantung, artinya kesalahan atau kualitas buruk pada tahap awal akan terakumulasi dan memperburuk hasil akhir.

Secara konseptual, pipeline ini dapat dibaca sebagai:

Dunia Nyata → Data Visual → Data Siap Olah → Informasi → Keputusan

1.2.1. Akuisisi Citra

(Image Acquisition)

Apa yang dimaksud?

Akuisisi citra adalah proses mengubah dunia fisik (analog) menjadi data digital yang dapat diproses komputer. Ini adalah gerbang awal seluruh sistem visi komputer.

Apa saja yang termasuk akuisisi?

  • Kamera RGB (webcam, kamera ponsel)
  • Kamera grayscale
  • Kamera multispektral / hyperspektral
  • Kamera thermal
  • Sensor depth (LiDAR, Kinect)

Faktor teknis penting

Pada tahap ini ditentukan:

  • Resolusi citra
  • Sudut pengambilan
  • Pencahayaan
  • Noise sensor
  • Frame rate (untuk video)

Kenapa tahap ini krusial?

  • Informasi yang tidak tertangkap kamera tidak bisa “diciptakan” kembali
  • Noise berlebih akan mempersulit semua tahap berikutnya
  • Salah pilih sensor → algoritma sehebat apa pun hasilnya buruk

Insight penting:

Banyak kegagalan sistem visi komputer bukan karena algoritma, tetapi karena akuisi data yang buruk.

1.2.2. Pengolahan Citra

(Image Processing / Preprocessing)

Tujuan utama

Menyiapkan citra agar:

  • Lebih bersih
  • Lebih konsisten
  • Lebih mudah dianalisis

Ini bukan tahap memahami isi citra, tetapi membuat data “layak dibaca”.

Operasi yang umum dilakukan

  • Konversi warna (RGB → Grayscale)
  • Filtering (noise reduction)
  • Normalisasi intensitas
  • Resize / cropping
  • Histogram equalization

Analogi sederhana

Jika citra adalah dokumen:

  • Akuisisi → memindai dokumen
  • Pengolahan → membersihkan noda, memperjelas teks

Kenapa preprocessing penting?

  • Mengurangi kompleksitas komputasi
  • Menstabilkan performa algoritma
  • Mengurangi variasi yang tidak relevan (noise, pencahayaan)

Insight penting:

Algoritma visi komputer tidak bekerja langsung pada citra mentah kamera.

1.2.3. Analisis Citra

(Image Analysis / Feature Extraction)

Apa yang dilakukan di sini?

Pada tahap ini sistem mulai mengambil informasi penting dari citra.

Fokus utamanya adalah:

  • Apa saja ciri (feature) yang mewakili objek
  • Bagian mana yang relevan, mana yang diabaikan

Contoh fitur yang diekstraksi

  • Tepi (edge)
  • Sudut (corner)
  • Tekstur
  • Bentuk
  • Distribusi warna
  • Kontur

Bentuk output

  • Bukan lagi citra
  • Melainkan vektor fitur atau representasi numerik

Contoh:

  • Dari gambar wajah → vektor angka
  • Dari tulisan → pola piksel yang mewakili karakter

Kenapa tahap ini kunci?

  • Di sinilah citra berubah dari data visual menjadi data numerik bermakna
  • Kualitas fitur menentukan keberhasilan klasifikasi atau deteksi

Insight akademik:

Pada deep learning, tahap ini tidak hilang, tetapi dipelajari otomatis oleh network.

1.2.4. Pemahaman Citra

(Image Understanding / Decision Making)

Apa yang dimaksud pemahaman citra?

Ini adalah tahap kognitif dari visi komputer:

  • Menghubungkan fitur dengan makna
  • Mengambil keputusan berbasis informasi visual

Contoh aktivitas di tahap ini

  • Klasifikasi (apa objeknya?)
  • Deteksi (di mana objeknya?)
  • Segmentasi (bagian mana objek?)
  • Pelacakan (bagaimana objek bergerak?)
  • Diagnosis / keputusan otomatis

Output akhir

  • Label
  • Keputusan
  • Aksi sistem

Contoh:

  • “Ini tumor ganas”
  • “Mobil di depan berhenti”
  • “Wajah dikenali sebagai user A”

Hubungan dengan AI

  • Tahap ini sering melibatkan:
    • Machine Learning
    • Deep Learning
    • Rule-based reasoning

Insight penting:

Visi komputer baru benar-benar ‘cerdas’ di tahap ini.

2. Sejarah Singkat Visi Komputer

  • 1960‑an: Awal riset pengenalan pola dan citra digital
  • 1970‑an: Deteksi tepi, segmentasi, dan teori vision (David Marr)
  • 1980–1990‑an: Pendekatan statistik dan geometris
  • 2000‑an: Machine learning dan kemunculan OpenCV
  • 2010–sekarang: Deep learning (CNN, YOLO, R‑CNN)

Evolusi ini menunjukkan pergeseran dari aturan manual ke pembelajaran berbasis data.

3. Aplikasi Visi Komputer di Dunia Nyata

3.1 Kesehatan

  • Analisis citra medis (X‑ray, MRI, CT Scan)
  • Deteksi kanker dan penyakit mata

3.2 Transportasi

  • Kendaraan otonom
  • Advanced Driver Assistance System (ADAS)
  • Pengenalan plat nomor

3.3 Industri

  • Inspeksi kualitas otomatis
  • Robotika dan kontrol produksi

3.4 Keamanan

  • Pengenalan wajah
  • Biometrik
  • Sistem pengawasan cerdas

4. Dasar Citra Digital

4.1 Definisi Citra Digital

Citra digital adalah representasi diskrit dari data visual dalam bentuk matriks piksel, di mana setiap piksel menyimpan nilai intensitas atau warna.

4.2 Karakteristik Citra

  • Resolusi: jumlah piksel
  • Kedalaman bit: rentang nilai intensitas
  • Kanal warna: grayscale (1 kanal), RGB (3 kanal)

4.3 Representasi Matematis

Citra dinyatakan sebagai fungsi:

f(x, y) → nilai intensitas / warna pada koordinat (x, y)

5. Model Warna

5.1 RGB

Model warna aditif berbasis cahaya (Red, Green, Blue)

5.2 Grayscale

Satu kanal intensitas (0–255), umum untuk preprocessing

Formula konversi: Gray = 0.299R + 0.587G + 0.114B

5.3 HSV

Memisahkan hue, saturation, dan value – lebih intuitif untuk pemrosesan warna

5.4 CMYK

Model subtraktif untuk percetakan

6. Operasi Dasar pada Citra Digital

Operasi dasar pada citra digital adalah fondasi dari seluruh algoritma visi komputer, baik klasik maupun modern. Hampir semua pipeline computer vision terdiri dari rangkaian operasi-operasi ini, baik secara eksplisit (algoritma klasik) maupun implisit (dalam layer CNN).

Secara umum, operasi dasar dibagi menjadi tiga kelompok besar:

  1. Transformasi Geometri
  2. Transformasi Intensitas
  3. Operasi Spasial

6.1 Transformasi Geometri

Konsep Utama

Transformasi geometri adalah operasi yang mengubah posisi spasial piksel, tetapi tidak mengubah nilai intensitasnya.

Secara matematis:

Nilai piksel tetap, koordinat piksel berubah

Transformasi ini bekerja pada domain spasial (x, y).

a. Translasi (Translation)

Definisi
Translasi adalah pergeseran seluruh citra sejauh vektor tertentu.

Jika citra awal:

  • Piksel berada di (x, y)

Setelah translasi (tx, ty):

  • Pindah ke (x′, y′) = (x + tx, y + ty)

Kegunaan nyata:

  • Koreksi pergeseran kamera
  • Registrasi citra (image alignment)
  • Augmentasi data

Catatan penting akademik:

  • Translasi tidak mengubah bentuk atau orientasi objek
  • Hanya mengubah lokasi relatif terhadap frame

b. Rotasi (Rotation)

Definisi
Rotasi memutar citra terhadap suatu titik pusat (biasanya pusat citra).

Kegunaan:

  • Normalisasi orientasi objek
  • Augmentasi data training
  • Koreksi sudut pengambilan gambar

Isu penting:

  • Interpolasi (nearest, bilinear, bicubic)
  • Rotasi besar dapat menyebabkan kehilangan piksel (cropping)

c. Scaling (Penskalaan)

Definisi
Scaling mengubah ukuran citra (memperbesar atau memperkecil).

Jenis scaling:

  • Upscaling (zoom in)
  • Downscaling (zoom out)

Kegunaan:

  • Penyesuaian resolusi input model
  • Fast processing (downscale dulu, proses, lalu refine)

Trade-off utama:

Scaling

Dampak

Downscale

Lebih cepat, detail hilang

Upscale

Detail semu, noise muncul

d. Flipping

Definisi
Pencerminan citra:

  • Horizontal
  • Vertikal

Kegunaan utama:

  • Data augmentation
  • Mengatasi bias orientasi data

Kesimpulan Transformasi Geometri

Digunakan untuk mengontrol posisi dan orientasi data, bukan memperbaiki kualitas visual.

6.2 Transformasi Intensitas

Konsep Utama

Transformasi intensitas mengubah nilai piksel, tetapi tidak mengubah posisinya.

Secara matematis:

Koordinat tetap, intensitas berubah

Beroperasi pada fungsi:

s=T(r)s = T(r)s=T(r)

di mana:

  • r = intensitas input
  • s = intensitas output

a. Brightness Adjustment

Definisi
Menambah atau mengurangi nilai intensitas piksel secara konstan.

I′=I+cI' = I + cI′=I+c

Efek:

  • c > 0 → citra lebih terang
  • c < 0 → citra lebih gelap

Masalah klasik:

  • Saturation → piksel mentok di 0 atau 255

b. Contrast Adjustment

Definisi
Mengatur jarak antar nilai intensitas.

I′=αI+βI' = \alpha I + \betaI′=αI+β

  • α: kontrol kontras
  • β: kontrol brightness

Kegunaan:

  • Menonjolkan objek
  • Memperbaiki citra low contrast

c. Gamma Correction

Definisi
Transformasi non-linear untuk mengoreksi respon visual.

I′=IγI' = I^\gammaI′=Iγ

Makna penting:

  • γ < 1 → memperjelas area gelap
  • γ > 1 → menekan area terang

Aplikasi nyata:

  • Kamera
  • Medical imaging
  • Display calibration

d. Histogram Equalization

Definisi
Menyebarkan nilai intensitas agar distribusinya lebih merata.

Tujuan:

  • Meningkatkan kontras global
  • Mengungkap detail tersembunyi

Catatan akademik:

  • Bisa memperkuat noise
  • Versi adaptif (CLAHE) lebih stabil

Kesimpulan Transformasi Intensitas

Digunakan untuk memperbaiki kualitas citra secara visual dan statistik, bukan struktur spasialnya.

6.3 Operasi Spasial

Konsep Utama

Operasi spasial mempertimbangkan hubungan antar piksel, biasanya menggunakan kernel (mask).

Secara matematis:

g(x,y)=∑∑f(x+i,y+j)⋅h(i,j)g(x, y) = \sum\sum f(x+i, y+j) \cdot h(i, j)g(x,y)=∑∑f(x+i,y+j)⋅h(i,j)

Ini adalah dasar dari konvolusi.

a. Filtering

Tujuan:

  • Menghaluskan (smoothing)
  • Menajamkan (sharpening)

Contoh filter:

  • Mean filter
  • Gaussian filter
  • Median filter

Trade-off utama:

  • Terlalu halus → detail hilang
  • Kurang halus → noise tersisa

b. Edge Detection

Definisi
Menemukan perubahan intensitas tajam (diskontinuitas).

Makna penting:

  • Edge ≈ batas objek

Operator umum:

  • Sobel
  • Prewitt
  • Canny

Kenapa edge penting?

  • Dasar segmentasi
  • Deteksi bentuk
  • Feature extraction

c. Operasi Morfologi

Beroperasi pada citra biner

Operasi utama:

  • Erosi → mengecilkan objek
  • Dilasi → membesarkan objek
  • Opening → hapus noise kecil
  • Closing → tutup celah kecil

Aplikasi:

  • Praproses OCR
  • Analisis bentuk
  • Refinement segmentasi

7. Python untuk Visi Komputer

7.1 Mengapa Python?

  • Mudah dipelajari
  • Ekosistem library kuat
  • Cocok untuk riset dan prototyping

7.2 Library Utama

  • OpenCV – algoritma visi komputer lengkap
  • PIL/Pillow – manipulasi citra sederhana
  • NumPy – operasi numerik dan array
  • Matplotlib – visualisasi

8. Analisis Citra Sederhana

8.1 Grayscale

Digunakan untuk menyederhanakan data dan mengurangi kompleksitas komputasi

8.2 Histogram

Menunjukkan distribusi intensitas piksel, berguna untuk evaluasi kualitas citra

8.3 Thresholding

Mengubah citra grayscale menjadi biner

  • Binary threshold
  • Adaptive threshold
  • Otsu threshold

9. Struktur Data Python dalam Visi Komputer

  • List: menyimpan kumpulan piksel atau hasil
  • Dictionary: metadata citra dan kanal
  • Tuple: ukuran citra, nilai warna

Pemahaman struktur data penting untuk membangun pipeline visi komputer yang rapi dan modular.
